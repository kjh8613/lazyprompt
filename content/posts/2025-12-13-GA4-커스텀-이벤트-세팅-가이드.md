## 🎯 프롬프트 설명
이 프롬프트는 구글 애널리틱스 4(GA4)에서 맞춤 이벤트를 설정하는 데 필요한 완벽한 가이드를 제공합니다. 초보자도 쉽게 따라 할 수 있도록 구글 태그 매니저(GTM) 연동 및 트리거 설정을 포함한 모든 단계를 상세하게 설명하여 데이터 분석 역량을 향상시킵니다.

## 📋 프롬프트 내용 (복사해서 사용하세요)
```markdown
# Role
당신은 구글 애널리틱스(GA4)와 구글 태그 매니저(GTM) 전문가이며, 웹사이트 분석 및 데이터 추적 분야에서 10년 이상의 경험을 가진 시니어 마케팅 분석가입니다. 당신의 임무는 초보자도 이해하기 쉬운 맞춤 이벤트 설정 가이드를 제공하는 것입니다.

# Context
사용자는 구글 애널리틱스 4(GA4)를 사용하여 웹사이트의 사용자 행동을 분석하고자 합니다. 하지만 기본적으로 수집되지 않는 특정 버튼 클릭, 스크롤 깊이, 폼 제출 등의 이벤트를 추적해야 하며, 이를 위해 구글 태그 매니저(GTM)를 활용하여 맞춤 이벤트를 설정하는 방법을 알고 싶어합니다. 사용자는 GA4 및 GTM에 대한 기본적인 이해는 있지만, 실제로 설정하는 단계에서 어려움을 겪고 있습니다.

# Task
다음 단계를 포함하여 GA4 맞춤 이벤트 설정 가이드를 제공하십시오. 모든 단계는 초보자가 이해하기 쉽게 설명해야 합니다.

1. **GA4 맞춤 이벤트 설정의 필요성**: 왜 맞춤 이벤트가 필요한지, 기본 이벤트와 어떤 차이가 있는지 간략하게 설명합니다. (2문장 이내)
2. **구글 태그 매니저(GTM) 계정 설정 및 GA4 연동**:
    * GTM 계정 생성 및 GA4 속성 연결 방법을 단계별로 설명합니다. (스크린샷 첨부 없이 텍스트로 설명)
    * GA4 구성 태그 설정 방법을 설명합니다. (스크린샷 첨부 없이 텍스트로 설명)
3. **특정 버튼 클릭 추적을 위한 맞춤 이벤트 설정**:
    * 추적할 버튼의 HTML 요소(ID, Class 등)를 찾는 방법을 설명합니다.
    * GTM에서 '태그' 생성 (태그 유형, 추적 유형, 이벤트 이름 설정 포함) 단계를 상세히 설명합니다.
    * GTM에서 '트리거' 생성 (트리거 유형, 이벤트 조건 설정 포함) 단계를 상세히 설명합니다.
    * 생성한 태그와 트리거를 연결하는 방법을 설명합니다.
    * GTM 변경 사항을 '미리보기' 모드로 테스트하는 방법을 설명합니다.
    * GTM 변경 사항을 '제출'하여 게시하는 방법을 설명합니다.
4. **스크롤 깊이 추적을 위한 맞춤 이벤트 설정**:
    * GTM에서 '새로운 변수'를 설정하여 스크롤 깊이를 캡처하는 방법을 설명합니다. (변수 유형, 설정 포함)
    * GTM에서 '태그' 생성 (태그 유형, 추적 유형, 이벤트 이름 설정 포함) 단계를 상세히 설명합니다.
    * GTM에서 '트리거' 생성 (트리거 유형, 이벤트 조건 설정 포함 - 예: 25%, 50%, 75%, 90% 스크롤) 단계를 상세히 설명합니다.
    * 생성한 태그와 트리거를 연결하는 방법을 설명합니다.
    * GTM 변경 사항을 '미리보기' 모드로 테스트하는 방법을 설명합니다.
    * GTM 변경 사항을 '제출'하여 게시하는 방법을 설명합니다.
5. **GA4에서 이벤트 확인**:
    * GA4 실시간 보고서에서 설정한 이벤트가 정상적으로 수집되는지 확인하는 방법을 설명합니다.
    * GA4 탐색 보고서를 활용하여 이벤트 데이터를 분석하는 기본적인 방법을 설명합니다. (예: 이벤트 수, 사용자 수 확인)
6. **주의 사항 및 문제 해결**:
    * 흔히 발생하는 문제 (예: 이벤트가 수집되지 않는 경우)에 대한 해결책을 제시합니다.
    * 맞춤 이벤트 설정 시 고려해야 할 중요한 점들을 강조합니다. (예: 이벤트 이름 규칙, 데이터 수집 제한 등)

# Constraints
* 가이드는 초보자를 대상으로 하며, 전문 용어 사용을 최소화하고 쉬운 단어로 설명해야 합니다.
* 모든 단계는 명확하고 간결하게 작성되어야 하며, 불필요한 내용은 포함하지 않습니다.
* 가이드의 어조는 친절하고 도움이 되는 느낌으로 유지해야 합니다.
* 스크린샷은 첨부하지 않고 텍스트로만 모든 단계를 설명해야 합니다.
* 예시 코드나 복잡한 설정은 최소화하고, 가장 기본적인 설정 방법을 중심으로 설명합니다.
* 설명은 반드시 순서대로 진행되어야 하며, 각 단계는 명확하게 구분되어야 합니다.

# Output Format
마크다운 형식으로 작성된 GA4 맞춤 이벤트 설정 가이드. 각 단계는 번호 매기기로 구분하고, 필요한 경우 소제목을 사용하여 가독성을 높입니다. 가이드의 마지막에는 "참고 자료" 섹션을 추가하여 추가 학습을 위한 링크를 제공합니다 (GA4 공식 문서, GTM 공식 문서).
```

## 💡 사용 팁
1. 위 프롬프트에서 추적하고 싶은 이벤트 종류 (버튼 클릭, 스크롤 깊이 외) 를 변경하여 맞춤형 가이드를 생성할 수 있습니다. 예를 들어 '폼 제출 추적' 또는 '동영상 재생 추적'으로 변경할 수 있습니다.
2. 웹사이트의 구조나 특정 버튼의 HTML 요소에 대한 정보를 추가적으로 제공하면, 더욱 정확하고 맞춤화된 가이드를 얻을 수 있습니다.
3. 이 프롬프트는 GPT-4 또는 Gemini Pro와 같은 고급 언어 모델에서 가장 효과적입니다.
