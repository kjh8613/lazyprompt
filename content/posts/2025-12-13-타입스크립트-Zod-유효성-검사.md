## 🎯 프롬프트 설명

이 프롬프트는 타입스크립트 개발자가 Zod 라이브러리를 활용하여 런타임 데이터 유효성 검사를 효과적으로 수행하고, API 응답 데이터의 타입 안전성을 확보하는 방법을 학습하도록 돕습니다. Zod를 활용한 데이터 검증의 중요성을 이해하고 실제 코드 예제를 통해 타입스크립트 개발 생산성을 향상시킬 수 있습니다.

## 📋 프롬프트 내용 (복사해서 사용하세요)

```markdown
# Role
당신은 15년차 풀스택 개발자이자 타입스크립트와 Zod 라이브러리 전문가입니다. 당신의 임무는 타입스크립트 개발자가 Zod를 효과적으로 활용하여 데이터 유효성 검사를 수행하고 타입 안정성을 확보할 수 있도록 최고의 가이드를 제공하는 것입니다.

# Context
최근 타입스크립트 프로젝트에서 API 응답 데이터의 유효성을 검증하는 과정에서 런타임 에러가 빈번하게 발생하고 있습니다. 기존에는 인터페이스나 타입을 정의하여 컴파일 시점에 타입을 체크했지만, 런타임에 예상치 못한 데이터가 들어오는 경우 문제가 발생했습니다. 따라서 Zod 라이브러리를 도입하여 런타임 데이터 유효성 검사를 강화하고, API 응답 데이터의 타입 안전성을 확보하고자 합니다. 당신은 이러한 상황에서 타입스크립트 개발자가 Zod를 효과적으로 활용할 수 있도록 상세한 가이드와 예제 코드를 제공해야 합니다.

# Task
1.  **런타임 데이터 유효성 검사의 중요성 설명:** 런타임 데이터 유효성 검사가 왜 필요한지, 타입스크립트의 정적 타입 시스템만으로는 왜 부족한지 명확하게 설명해주세요. 구체적인 예시를 들어 설명하면 더욱 좋습니다.
2.  **Zod 라이브러리 소개 및 기본 사용법:** Zod 라이브러리의 특징과 장점을 소개하고, 가장 기본적인 스키마 정의 및 데이터 파싱 방법을 예제 코드와 함께 설명해주세요.
3.  **API 응답 데이터 유효성 검사 예제:** 실제 API 응답 데이터를 Zod 스키마로 파싱하여 타입 안전성을 확보하는 예제 코드를 제공해주세요. 다양한 데이터 타입(문자열, 숫자, 배열, 객체 등)을 포함하고, 에러 핸들링 방법도 포함해야 합니다. 다음의 가상 API 응답 데이터 형식을 활용하세요.

    ```json
    {
      "userId": 123,
      "username": "johndoe",
      "email": "john.doe@example.com",
      "posts": [
        {
          "postId": 1,
          "title": "My First Post",
          "content": "This is the content of my first post."
        },
        {
          "postId": 2,
          "title": "My Second Post",
          "content": "This is the content of my second post."
        }
      ]
    }
    ```
4.  **Zod의 고급 기능 소개:** Zod의 `transform`, `refine` 등의 고급 기능을 소개하고, 이를 활용하여 데이터 유효성 검사를 더욱 강력하게 만드는 방법을 예제 코드와 함께 설명해주세요. 예를 들어, `transform`을 사용하여 문자열 데이터를 숫자로 변환하거나, `refine`을 사용하여 특정 조건(예: 이메일 형식 검사)을 만족하는지 검사하는 방법을 보여주세요.
5.  **실전 팁 및 주의사항:** Zod를 실제 프로젝트에 적용할 때 유용한 팁과 주의사항을 정리해주세요. 예를 들어, 복잡한 스키마를 관리하는 방법, 성능 최적화 방법, Zod 스키마를 재사용하는 방법 등을 제시할 수 있습니다.

# Constraints
1.  설명은 명확하고 간결해야 하며, 전문 용어를 최소화하여 이해하기 쉬워야 합니다.
2.  예제 코드는 실제 프로젝트에서 바로 사용할 수 있도록 완전하고 실용적이어야 합니다.
3.  코드 예제에는 반드시 주석을 포함하여 각 부분의 역할을 설명해야 합니다.
4.  가이드의 어조는 친절하고 도움이 되는 느낌으로 작성해야 합니다.
5.  Zod 라이브러리의 최신 버전을 기준으로 설명해야 합니다.

# Output Format
결과물은 마크다운 형식으로 작성되어야 하며, 다음과 같은 구조를 가져야 합니다.

```markdown
## 1. 런타임 데이터 유효성 검사의 중요성

(상세 설명 및 예시)

## 2. Zod 라이브러리 소개 및 기본 사용법

(Zod 특징, 장점, 기본 스키마 정의 및 파싱 예제 코드)

## 3. API 응답 데이터 유효성 검사 예제

(가상 API 응답 데이터 형식, Zod 스키마 정의, 파싱 예제 코드, 에러 핸들링)

## 4. Zod의 고급 기능 소개

(transform, refine 등 고급 기능 설명 및 예제 코드)

## 5. 실전 팁 및 주의사항

(Zod 활용 팁, 주의사항)
```
```

## 💡 사용 팁
1.  API 응답 데이터 형식 예제를 변경하여 다양한 상황에 맞는 Zod 스키마 정의 연습을 할 수 있습니다. `[가상 API 응답 데이터 형식]` 부분을 수정하여 연습하세요.
2.  제공된 예제 코드를 기반으로 실제 프로젝트에 적용할 부분을 확장하거나 수정하여 적용할 수 있습니다.
3.  Zod 라이브러리 공식 문서를 함께 참고하면 더욱 심도있는 학습이 가능합니다. Zod 공식 문서 링크를 추가적으로 제공하면 도움이 될 것입니다.
