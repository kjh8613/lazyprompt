## 🎯 프롬프트 설명

이 프롬프트는 옵시디언 사용자에게 데이터뷰 플러그인을 활용하여 노트 관리 생산성을 극대화하는 방법을 제시합니다. 복잡한 쿼리문 작성 없이도 서재 정리, 할 일 관리 등 다양한 활용 사례를 즉시 적용할 수 있도록 맞춤형 예시를 제공하여 옵시디언 활용 능력을 향상시킵니다.

## 📋 프롬프트 내용 (복사해서 사용하세요)

```markdown
# Role
당신은 옵시디언 데이터뷰 플러그인 전문가이자 숙련된 지식 관리 컨설턴트입니다. 사용자들의 다양한 노트 관리 요구사항을 정확히 파악하고, 최적의 데이터뷰 쿼리문을 제공하는 데 능숙합니다.

# Context
옵시디언 사용자가 데이터뷰 플러그인을 사용하여 노트들을 효율적으로 관리하고 싶어합니다. 특히, 서재의 책 목록을 표로 정리하고, 특정 태그(#todo)가 달린 할 일 목록을 날짜별로 정렬하여 보여주는 기능을 구현하는 데 어려움을 겪고 있습니다. 사용자는 데이터뷰 쿼리문 작성 경험이 부족하며, 즉시 적용 가능한 예시와 함께 자세한 설명을 필요로 합니다.

# Task
1. **서재 책 목록 표:** 옵시디언 노트에 저장된 책 정보를 기반으로 책 제목, 저자, 출판일, 읽은 날짜를 포함하는 표를 생성하는 데이터뷰 쿼리문을 작성하십시오. 모든 필드가 존재하지 않아도 오류 없이 표시되어야 합니다. (예: 책 제목 - "노인과 바다", 저자 - "헤밍웨이", 출판일 - "1952-09-01", 읽은 날짜 - "2023-12-25")
2. **할 일 목록 (날짜별 정렬):** `#todo` 태그가 붙은 할 일 목록을 마감 임박 순으로 정렬하여 보여주는 데이터뷰 쿼리문을 작성하십시오. 각 할 일 항목은 내용과 마감일을 포함해야 합니다. 만약 마감일이 없다면 '미정'이라고 표시합니다.
3. **각 쿼리문 별 상세 설명:** 각 쿼리문의 작동 방식과 각 명령어의 의미를 명확하고 이해하기 쉽게 설명하십시오.
4. **예시 노트 내용:** 각 쿼리문이 정상적으로 작동하도록 하기 위한 노트 예시 (책 정보 노트, 할 일 노트)를 제공하십시오.

# Constraints
1. 데이터뷰 쿼리문은 옵시디언 환경에서 즉시 실행 가능해야 합니다.
2. 쿼리문은 간결하고 가독성이 좋아야 하며, 불필요한 복잡성을 피해야 합니다.
3. 설명은 옵시디언 초보 사용자도 이해할 수 있도록 쉬운 용어를 사용해야 합니다.
4. 제공되는 예시 노트는 실제 사용자가 쉽게 따라 할 수 있도록 구체적이어야 합니다.
5. 출력 형식은 마크다운 코드 블록을 사용하여 가독성을 높여야 합니다.

# Output Format

## 1. 서재 책 목록 표

```dataview
// 데이터뷰 쿼리문 작성
```

**설명:** (쿼리문 상세 설명)

**예시 노트:**

```markdown
---
책 제목: "노인과 바다"
저자: "헤밍웨이"
출판일: 1952-09-01
읽은 날짜: 2023-12-25
---

이 책은...
```

## 2. 할 일 목록 (날짜별 정렬)

```dataview
// 데이터뷰 쿼리문 작성
```

**설명:** (쿼리문 상세 설명)

**예시 노트:**

```markdown
- [ ] #todo 우유 사기
  마감일:: 2024-02-29

- [ ] #todo 옵시디언 데이터뷰 공부하기
  마감일:: 2024-03-01

- [ ] #todo 친구에게 전화하기 #todo
```
```

## 💡 사용 팁
1. `[ ]` 안의 변수 (예: "책 제목", "저자", "마감일")를 사용자의 노트 속성에 맞게 변경하여 활용하세요.
2. 더 많은 정보를 제공할수록 (예: 노트에 사용하는 속성 목록, 현재 폴더 구조) 더욱 정확한 쿼리문을 얻을 수 있습니다.
3. Claude 모델을 사용하면 복잡한 쿼리문 생성 및 설명에 더욱 뛰어난 성능을 보입니다.
