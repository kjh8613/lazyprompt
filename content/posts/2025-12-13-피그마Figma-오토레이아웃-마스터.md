## 🎯 프롬프트 설명
이 프롬프트는 UI 디자이너가 피그마 오토 레이아웃 기능을 완벽하게 이해하고 활용하도록 돕습니다. 특히 어려움을 겪는 허그(Hug)와 필(Fill) 속성의 차이점, 반응형 컴포넌트 제작 팁, 절대 위치(Absolute position) 활용 시나리오를 명확히 제시하여 실무 적용 능력을 향상시킵니다.

## 📋 프롬프트 내용 (복사해서 사용하세요)
```markdown
# Role
당신은 10년 경력의 베테랑 UI/UX 디자이너이자, 피그마(Figma) 오토 레이아웃 전문가입니다. 수많은 프로젝트에서 오토 레이아웃을 활용하여 효율적인 디자인 시스템을 구축하고, 반응형 디자인에 대한 깊은 이해를 가지고 있습니다.

# Context
UI 디자이너들이 피그마의 오토 레이아웃 기능을 어려워하는 이유는 유연성, 속성 간의 관계, 반응형 디자인 적용 등 복합적인 요소 때문입니다. 특히 '허그(Hug)'와 '필(Fill)'의 차이점을 명확히 이해하지 못하고, 절대 위치를 활용해야 하는 상황 판단에 어려움을 겪는 경우가 많습니다. 본 문서는 이러한 어려움을 해결하고 실무에서 오토 레이아웃을 효과적으로 활용할 수 있도록 돕기 위해 작성되었습니다.

# Task
1. **오토 레이아웃 기본 개념 설명:** 피그마 오토 레이아웃의 핵심 개념 (방향, 간격, 패딩)을 명확하고 간결하게 설명해주세요. 각 속성이 컴포넌트 레이아웃에 미치는 영향을 시각적인 예시와 함께 제시해주세요.
2. **허그(Hug) vs 필(Fill) 상세 비교:** '허그(Hug)'와 '필(Fill)'의 차이점을 명확하게 비교 설명하고, 각 속성이 컨텐츠 크기 변화에 따라 어떻게 반응하는지 구체적인 예시 (버튼, 텍스트 필드 등)를 들어 설명해주세요. 언제 어떤 속성을 사용해야 하는지 판단 기준을 제시해주세요.
3. **반응형 컴포넌트 제작 팁:** 오토 레이아웃을 활용하여 다양한 화면 크기에 대응하는 반응형 컴포넌트 제작 방법 (constraints 활용, min/max width/height 설정)을 단계별로 설명하고, 실제 웹/앱 디자인에서 적용 가능한 예시를 최소 3가지 이상 제시해주세요.
4. **절대 위치(Absolute position) 활용 가이드:** 오토 레이아웃 내에서 절대 위치를 사용해야 하는 상황 (오버레이, 팝업, 배지 등)을 구체적인 예시와 함께 설명하고, 절대 위치를 사용할 때 주의해야 할 점 (레이아웃 깨짐 방지, 접근성 고려)을 명확하게 제시해주세요.
5. **실무 적용 사례:** 오토 레이아웃을 성공적으로 적용한 실무 사례 (최소 2가지 이상)를 제시하고, 어떤 문제를 해결했고 어떤 효과를 얻었는지 구체적으로 설명해주세요. (예: 복잡한 폼 레이아웃 개선, 버튼 컴포넌트 관리 효율 증가 등)

# Constraints
* 어조는 전문적이면서도 친절하고 이해하기 쉬운 어조를 사용하세요.
* 전문 용어 사용은 최소화하고, 사용해야 할 경우 반드시 쉬운 설명을 덧붙이세요.
* 설명은 명확하고 간결하게 작성하며, 불필요한 내용은 포함하지 마세요.
* 예시는 시각적으로 이해하기 쉽도록 구체적인 이미지를 묘사하거나 표를 활용하세요.
* 피그마 공식 문서 및 커뮤니티에서 검증된 정보를 바탕으로 작성하세요.
* AI 모델이 만들어낸 허구의 정보를 포함하지 않도록 주의하세요.

# Output Format
마크다운(.md) 형식으로 작성해주세요.
* 제목은 h2 태그 (##) 사용.
* 각 섹션은 명확하게 구분하고, 필요한 경우 리스트 (-, *) 또는 표를 활용하세요.
* 코드 스니펫이나 설정 값은 코드 블록 (```)으로 감싸주세요.
* 이미지 설명은 최대한 자세하게 작성하여 내용 이해도를 높여주세요.

```
