# React 컴포넌트 생성기: 모달(Modal) 컴포넌트 만들기

리액트(React)는 사용자 인터페이스(UI)를 구축하기 위한 강력하고 유연한 JavaScript 라이브러리입니다. 리액트의 핵심은 재사용 가능한 컴포넌트를 활용하여 복잡한 애플리케이션을 관리하고 개발하는 데 있습니다. 이 글에서는 특정 기능을 수행하는 리액트 컴포넌트, 그 중에서도 **모달(Modal) 컴포넌트**를 만들고, 그 코드와 함께 자세한 설명을 제공합니다. 더불어, 스타일링을 위한 CSS 및 Tailwind CSS 클래스 예시도 함께 제공하여 실질적인 개발에 도움이 되도록 구성했습니다.

## 서론: 왜 모달 컴포넌트인가?

모달은 웹 애플리케이션에서 사용자에게 중요한 정보나 액션을 제공하기 위해 널리 사용되는 UI 패턴입니다. 사용자의 현재 흐름을 방해하지 않으면서, 필요한 정보를 보여주거나 특정 작업을 수행하도록 유도할 수 있습니다. 예를 들어, 사용자 계정 생성, 로그인, 세부 정보 확인, 경고 메시지 표시 등 다양한 상황에서 활용됩니다.

모달 컴포넌트를 효과적으로 만드는 것은 UI/UX 디자인의 핵심 요소 중 하나이며, 재사용 가능하고 유지보수가 용이한 코드를 작성하는 데 중요한 역할을 합니다.  이 글에서는 함수형 컴포넌트를 사용하여 모달을 만들고, 상태 관리, 스타일링, 그리고 접근성을 고려하여 실질적인 활용 가능성을 높입니다.

## 본론: 리액트 모달 컴포넌트 구현

### 1. 기본 구조

```jsx
import React, { useState, useEffect, useRef } from 'react';

// 모달 컴포넌트
const Modal = ({ isOpen, onClose, children, title }) => {
  // isOpen: 모달의 열림/닫힘 상태를 제어하는 prop (boolean)
  // onClose: 모달을 닫는 함수 (function)
  // children: 모달 내용 (ReactNode)
  // title: 모달 제목 (string)

  const [isVisible, setIsVisible] = useState(isOpen); // 모달의 visibility를 관리하는 state
  const modalRef = useRef(null); // 모달 영역 외부 클릭 감지를 위한 ref

  useEffect(() => {
    setIsVisible(isOpen); // isOpen prop이 변경될 때 visibility state 업데이트
  }, [isOpen]);

  useEffect(() => {
    // 외부 클릭 감지 로직
    const handleClickOutside = (event) => {
      if (modalRef.current && !modalRef.current.contains(event.target)) {
        onClose(); // 모달 외부 클릭 시 닫기 함수 호출
      }
    };

    if (isOpen) {
      document.addEventListener('mousedown', handleClickOutside);
    } else {
      document.removeEventListener('mousedown', handleClickOutside);
    }

    return () => {
      document.removeEventListener('mousedown', handleClickOutside); // 컴포넌트 언마운트 시 이벤트 리스너 제거
    };
  }, [isOpen, onClose]);

  const handleClose = () => {
    onClose(); // 닫기 버튼 클릭 시 닫기 함수 호출
  };

  // 모달이 닫히는 애니메이션을 구현하기 위해 opacity를 조절
  const modalStyle = {
    opacity: isVisible ? 1 : 0,
    visibility: isVisible ? 'visible' : 'hidden',
    transition: 'opacity 0.3s ease-in-out, visibility 0.3s ease-in-out',
  };

  if (!isOpen && !isVisible) {
    return null; // 모달이 닫혀있고 애니메이션이 완료되면 null 반환 (렌더링 X)
  }

  return (
    // 백드롭 (Modal 배경)
    <div
      className="fixed top-0 left-0 w-full h-full bg-black bg-opacity-50 flex items-center justify-center z-50"
      style={modalStyle} // 인라인 스타일 적용
    >
      {/* 모달 내용 컨테이너 */}
      <div
        className="bg-white rounded-lg shadow-lg p-6 relative w-full max-w-md"
        ref={modalRef} // ref 연결
      >
        {/* 닫기 버튼 */}
        <button
          className="absolute top-2 right-2 text-gray-500 hover:text-gray-700"
          onClick={handleClose} // 닫기 버튼 클릭 이벤트
        >
          <svg
            className="h-6 w-6"
            fill="none"
            stroke="currentColor"
            viewBox="0 0 24 24"
            xmlns="http://www.w3.org/2000/svg"
          >
            <path
              strokeLinecap="round"
              strokeLinejoin="round"
              strokeWidth="2"
              d="M6 18L18 6M6 6l12 12"
            ></path>
          </svg>
        </button>

        {/* 모달 제목 */}
        {title && <h2 className="text-xl font-semibold mb-4">{title}</h2>}

        {/* 모달 내용 */}
        <div>{children}</div>
      </div>
    </div>
  );
};

export default Modal;
```

**코드 설명:**

*   **`useState` 훅:** `isVisible` 상태를 사용하여 모달의 가시성을 관리합니다. `isOpen` prop이 변경될 때 `useEffect`를 통해 `isVisible`을 업데이트합니다.  애니메이션 효과를 위해 직접적으로 `isOpen`을 사용하지 않고 `isVisible`을 사용합니다.
*   **`useEffect` 훅:** 컴포넌트가 마운트/언마운트될 때 외부 클릭 감지 이벤트 리스너를 추가/제거합니다. `isOpen` 상태에 따라 이벤트 리스너를 관리하여 불필요한 이벤트 핸들링을 방지합니다.
*   **`useRef` 훅:** 모달 컨테이너에 대한 참조를 생성하여 모달 외부 클릭을 감지하는 데 사용합니다.
*   **`modalStyle`:**  CSS transition을 사용하여 모달이 나타나고 사라질 때 부드러운 애니메이션 효과를 제공합니다.  `opacity`와 `visibility`를 함께 사용하여 애니메이션을 구현합니다.
*   **`handleClose`:** 모달 닫기 버튼 클릭 시 호출되는 함수입니다. `onClose` prop을 통해 부모 컴포넌트에 닫힘 이벤트를 전달합니다.
*   **Tailwind CSS 클래스:** 모달의 스타일링을 위해 Tailwind CSS 클래스를 사용했습니다.  `fixed`, `top-0`, `left-0`, `w-full`, `h-full`, `bg-black`, `bg-opacity-50`, `flex`, `items-center`, `justify-center`, `z-50`, `bg-white`, `rounded-lg`, `shadow-lg`, `p-6`, `relative`, `w-full`, `max-w-md`, `absolute`, `top-2`, `right-2`, `text-gray-500`, `hover:text-gray-700`, `h-6`, `w-6`, `text-xl`, `font-semibold`, `mb-4` 등 다양한 클래스를 활용하여 모달의 레이아웃, 배경, 그림자, 폰트 등을 스타일링했습니다.
*   **접근성:** WAI-ARIA 속성을 추가하여 스크린 리더 사용자에게 모달의 역할을 알릴 수 있습니다 (예: `aria-modal="true"`).  현재 코드에는 포함되어 있지 않지만, 실제 프로덕션 환경에서는 접근성을 고려해야 합니다.

### 2. 사용 예시

```jsx
import React, { useState } from 'react';
import Modal from './Modal'; // 모달 컴포넌트 import

const App = () => {
  const [isModalOpen, setIsModalOpen] = useState(false); // 모달 열림/닫힘 상태 관리

  const openModal = () => {
    setIsModalOpen(true); // 모달 열기
  };

  const closeModal = () => {
    setIsModalOpen(false); // 모달 닫기
  };

  return (
    <div>
      <button onClick={openModal}>Open Modal</button> {/* 모달 열기 버튼 */}

      {/* 모달 컴포넌트 사용 */}
      <Modal
        isOpen={isModalOpen} // 열림/닫힘 상태 전달
        onClose={closeModal} // 닫기 함수 전달
        title="Modal Title" // 모달 제목
      >
        {/* 모달 내용 */}
        <p>This is the modal content.</p>
        <button onClick={closeModal}>Close</button> {/* 모달 내부 닫기 버튼 */}
      </Modal>
    </div>
  );
};

export default App;
```

**코드 설명:**

*   `App` 컴포넌트에서 `Modal` 컴포넌트를 사용합니다.
*   `isModalOpen` 상태를 사용하여 모달의 열림/닫힘 상태를 관리합니다.
*   `openModal` 함수는 모달을 열고, `closeModal` 함수는 모달을 닫습니다.
*   `Modal` 컴포넌트에 `isOpen`, `onClose`, `title`, `children` prop을 전달합니다.

이 CSS는 모달 오버레이, 모달 콘텐츠, 닫기 버튼의 기본 스타일을 정의합니다. 필요에 따라 스타일을 조정하여 애플리케이션의 디자인에 맞게 커스터마이징할 수 있습니다. Tailwind CSS를 사용하는 경우, 이 CSS 코드는 Tailwind CSS 클래스로 대체될 수 있습니다.

## 결론: 재사용 가능한 모달 컴포넌트의 가치

이 글에서는 리액트 함수형 컴포넌트를 사용하여 모달 컴포넌트를 만드는 방법을 자세히 설명했습니다.  `useState`, `useEffect`, `useRef` 훅을 활용하여 모달의 상태 관리, 외부 클릭 감지, 애니메이션 효과를 구현했으며, Tailwind CSS 클래스와 CSS 예시를 통해 스타일링 방법을 제시했습니다.

이러한 재사용 가능한 모달 컴포넌트는 웹 애플리케이션의 UI를 개선하고, 개발 생산성을 높이는 데 기여합니다.  뿐만 아니라, 접근성을 고려하여 모든 사용자가 편리하게 이용할 수 있도록 하는 것이 중요합니다.

이 글에서 제공된 코드를 기반으로, 필요에 따라 모달 컴포넌트를 커스터마이징하고 확장하여 더욱 강력하고 유연한 UI를 구축할 수 있습니다. 앞으로도 리액트 컴포넌트 개발에 대한 학습을 지속하고, 다양한 UI 패턴을 익혀 더욱 효율적인 웹 애플리케이션을 만들어나가기를 바랍니다.
