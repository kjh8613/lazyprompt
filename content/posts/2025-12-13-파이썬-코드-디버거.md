## 파이썬 코드 디버깅: 초보자를 위한 친절한 안내와 성능 개선 팁

파이썬은 배우기 쉽고 강력한 프로그래밍 언어이지만, 코드를 작성하다 보면 필연적으로 버그와 마주하게 됩니다. 버그는 예상치 못한 동작을 일으키고, 프로그램의 실행을 멈추게 하는 주범이죠. 특히 초보 개발자에게는 이러한 버그를 발견하고 수정하는 과정이 어렵게 느껴질 수 있습니다.

이 글에서는 흔히 발생하는 파이썬 코드 버그를 예시로 살펴보고, 디버깅 방법을 초보자도 이해하기 쉽게 설명합니다. 또한, 코드의 성능을 개선할 수 있는 실용적인 팁도 함께 제공하여 파이썬 개발 실력을 한 단계 끌어올릴 수 있도록 돕겠습니다.

### 본론: 파이썬 코드 디버깅 실전 가이드

**1. 예제 코드:**

다음은 간단한 숫자 리스트의 평균을 계산하는 파이썬 코드입니다. 하지만 이 코드에는 버그가 숨어있습니다.

```python
def calculate_average(numbers):
    """숫자 리스트의 평균을 계산합니다."""
    total = 0
    for number in numbers:
        total += number
    average = total / len(numbers)
    return average

numbers = [1, 2, 3, 4, 5]
average = calculate_average(numbers)
print(f"평균: {average}")

numbers = [] # 빈 리스트를 넣으면 어떻게 될까요?
average = calculate_average(numbers)
print(f"평균: {average}")
```

**2. 버그 발생 및 원인 분석:**

위 코드를 실행하면 처음에는 정상적으로 평균을 출력하지만, 두 번째 `numbers` 리스트가 비어있을 때 `ZeroDivisionError: division by zero` 에러가 발생합니다.

**에러 메시지 분석:** `ZeroDivisionError: division by zero`는 0으로 나눌 수 없다는 뜻입니다.

**원인 분석:** `calculate_average` 함수에서 `average = total / len(numbers)` 부분에서 `len(numbers)`가 0이 되면 0으로 나누는 연산이 수행되어 에러가 발생합니다.

**3. 디버깅 방법:**

*   **Print 문 활용:** 문제 발생 가능성이 있는 부분에 `print()` 함수를 사용하여 변수의 값을 출력해봅니다. 예를 들어, `len(numbers)` 값이 0인지 확인하기 위해 `print(f"리스트 길이: {len(numbers)}")`를 추가할 수 있습니다.
*   **디버거 사용:** 파이썬에는 내장 디버거(`pdb`)와 다양한 IDE 디버거가 있습니다. 디버거를 사용하면 코드 실행을 단계별로 추적하고, 변수의 값을 실시간으로 확인할 수 있습니다. `pdb` 사용법은 다음과 같습니다.

    ```python
    import pdb

    def calculate_average(numbers):
        """숫자 리스트의 평균을 계산합니다."""
        total = 0
        for number in numbers:
            total += number
        pdb.set_trace() # 이 부분에서 실행이 멈춥니다.
        average = total / len(numbers)
        return average

    numbers = [1, 2, 3, 4, 5]
    average = calculate_average(numbers)
    print(f"평균: {average}")

    numbers = []
    average = calculate_average(numbers)
    print(f"평균: {average}")
    ```

    위 코드에서 `pdb.set_trace()`를 삽입하면 해당 라인에서 실행이 멈추고, 디버거 프롬프트가 나타납니다. 여기서 `n`을 누르면 다음 라인으로 진행하고, `p 변수명`을 입력하면 해당 변수의 값을 확인할 수 있습니다. `c`를 누르면 디버거를 종료하고 코드가 계속 실행됩니다.

**4. 버그 수정:**

`calculate_average` 함수에 `numbers` 리스트가 비어있는지 확인하는 조건을 추가하여 에러를 방지할 수 있습니다.

```python
def calculate_average(numbers):
    """숫자 리스트의 평균을 계산합니다."""
    if not numbers:
        return 0  # 빈 리스트일 경우 0을 반환
    total = 0
    for number in numbers:
        total += number
    average = total / len(numbers)
    return average

numbers = [1, 2, 3, 4, 5]
average = calculate_average(numbers)
print(f"평균: {average}")

numbers = []
average = calculate_average(numbers)
print(f"평균: {average}")
```

**5. 성능 개선 팁:**

*   **내장 함수 활용:** 파이썬은 다양한 내장 함수를 제공합니다. `sum()` 함수를 사용하면 리스트의 합을 더 효율적으로 계산할 수 있습니다.

    ```python
    def calculate_average(numbers):
        """숫자 리스트의 평균을 계산합니다."""
        if not numbers:
            return 0
        average = sum(numbers) / len(numbers)
        return average
    ```

*   **List Comprehension 활용:** 복잡한 반복문을 List Comprehension을 사용하여 더 간결하고 빠르게 만들 수 있습니다.

    ```python
    # 일반적인 반복문
    squares = []
    for i in range(10):
        squares.append(i**2)

    # List Comprehension
    squares = [i**2 for i in range(10)]
    ```

*   **적절한 자료 구조 선택:** 문제에 맞는 최적의 자료 구조를 선택하는 것은 성능에 큰 영향을 미칩니다. 예를 들어, 검색 속도가 중요한 경우 리스트 대신 딕셔너리나 집합을 사용하는 것이 좋습니다.

*   **Profiling:** `cProfile`과 같은 프로파일링 도구를 사용하여 코드의 병목 지점을 파악하고 최적화할 수 있습니다.

**6. 다른 흔한 버그 종류 및 디버깅 팁:**

*   **`NameError`**: 정의되지 않은 변수를 사용하려고 할 때 발생합니다. 변수 이름을 정확히 입력했는지, 변수가 사용되기 전에 초기화되었는지 확인해야 합니다.
*   **`TypeError`**: 잘못된 자료형의 연산을 수행하려고 할 때 발생합니다. 변수의 자료형을 확인하고, 필요한 경우 형 변환을 수행해야 합니다.
*   **`IndexError`**: 리스트 또는 튜플의 범위를 벗어난 인덱스에 접근하려고 할 때 발생합니다. 인덱스가 유효한 범위 내에 있는지 확인해야 합니다.
*   **`ValueError`**: 함수에 예상치 못한 값(자료형은 맞지만)을 전달할 때 발생합니다. 함수의 요구 사항을 충족하는 값을 전달해야 합니다.
*   **들여쓰기 오류 (IndentationError)**: 파이썬은 들여쓰기로 코드 블록을 구분합니다. 들여쓰기가 올바르게 되어 있는지 확인해야 합니다.
