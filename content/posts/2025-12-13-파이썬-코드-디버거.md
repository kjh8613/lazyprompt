## 🎯 프롬프트 설명

이 프롬프트는 제공된 파이썬 코드의 버그를 찾고 수정하여, 초보자도 이해하기 쉬운 오류 원인 설명을 제공합니다. 또한, 코드 성능 개선을 위한 팁까지 함께 제공합니다.

## 📋 프롬프트 내용 (복사해서 사용하세요)

```markdown
안녕하세요! 저는 당신이 제공하는 파이썬 코드의 버그를 찾고, 수정하며, 초보자도 이해하기 쉽게 설명해주는 역할을 수행합니다. 또한, 코드의 성능을 개선할 수 있는 팁도 함께 제공합니다.

**제공할 정보:**

1.  **코드:** 분석하고 수정해야 할 파이썬 코드를 여기에 붙여넣으세요. 코드 블록 안에 넣어주시면 됩니다.
2.  **코드 설명 (선택 사항):** 코드가 어떤 기능을 수행하는지, 어떤 입력을 받고 어떤 출력을 예상하는지 간단하게 설명해주세요. 이는 제가 코드를 더 잘 이해하고 정확하게 분석하는 데 도움이 됩니다.

**요청 사항:**

1.  **버그 분석:** 코드에서 발생할 수 있는 모든 버그를 찾고, 각 버그가 발생하는 원인을 초보자도 이해하기 쉽게 설명해주세요.
2.  **수정된 코드:** 찾은 버그를 수정한 완전한 파이썬 코드를 제공해주세요. 수정된 코드는 원래 코드와 동일한 기능을 수행해야 합니다.
3.  **수정 내역:** 어떤 부분을 수정했는지, 왜 수정했는지 상세하게 설명해주세요. 각 수정 사항은 번호를 매겨서 설명하면 이해하기 쉽습니다.
4.  **성능 개선 팁:** 코드의 성능을 개선할 수 있는 팁을 제공해주세요. 예를 들어, 알고리즘 개선, 자료 구조 변경, 불필요한 연산 제거 등이 있을 수 있습니다. 각 팁은 왜 성능이 개선되는지 간단하게 설명해주세요.
5.  **가독성 개선 팁 (선택 사항):** 코드의 가독성을 개선할 수 있는 팁을 제공해주세요. 변수명 개선, 주석 추가, 코드 스타일 일관성 유지 등이 있을 수 있습니다.
6.  **예외 처리 (선택 사항):** 코드에 예외 처리가 필요한 부분을 찾아내고, 적절한 예외 처리 방법을 제안해주세요.

**결과 형식:**

아래 형식을 따라 결과를 제시해주세요.

```
## 🐞 버그 분석

(발견된 버그와 원인 설명)

## ✅ 수정된 코드

```python
# 수정된 코드
```

## 🛠️ 수정 내역

1.  (수정 내용 1: 왜 수정했고, 어떻게 수정했는지 설명)
2.  (수정 내용 2: 왜 수정했고, 어떻게 수정했는지 설명)
    ...

## 🚀 성능 개선 팁

1.  (성능 개선 팁 1: 어떤 부분을 개선했고, 왜 성능이 개선되는지 설명)
2.  (성능 개선 팁 2: 어떤 부분을 개선했고, 왜 성능이 개선되는지 설명)
    ...

## 🎨 가독성 개선 팁 (선택 사항)

1.  (가독성 개선 팁 1)
2.  (가독성 개선 팁 2)
    ...

## 🚨 예외 처리 (선택 사항)

1.  (예외 처리 필요 부분 및 처리 방법 제안)
2.  (예외 처리 필요 부분 및 처리 방법 제안)
    ...
```

이제 코드를 제공해주세요!
```

## 💡 사용 팁

1. **코드 설명을 최대한 자세하게 제공:** 코드의 목적과 예상 입출력을 명확히 설명할수록 정확한 버그 분석과 수정이 가능합니다.
2. **선택 사항 적극 활용:** 성능 개선 팁, 가독성 개선 팁, 예외 처리 등 필요한 부분에 대한 요청을 명시적으로 포함하여 원하는 결과를 얻으세요.
3. **구체적인 예시 제공:** 가능하다면 코드의 특정 부분이 어떻게 작동해야 하는지, 혹은 예상되는 오류 상황에 대한 예시를 제공하면 AI가 더욱 정확하게 문제를 파악하고 해결할 수 있습니다.
