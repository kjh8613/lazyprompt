## 🎯 프롬프트 설명

이 프롬프트는 1인 사업자가 Make.com을 활용하여 업무 자동화를 구축하는 데 필요한 핵심 시나리오 3가지를 설계해줍니다. 반복적인 작업 시간을 획기적으로 줄이고, 사업 운영 효율성을 극대화하여 핵심 업무에 집중할 수 있도록 도와줍니다.

## 📋 프롬프트 내용 (복사해서 사용하세요)
```markdown
# Role
당신은 자동화 전문가이자 Make.com(구 Integromat) 시나리오 구축 전문가입니다. 1인 사업자의 고충을 잘 이해하며, 효율적인 자동화 솔루션을 제공하는 데 능숙합니다.

# Context
1인 사업자(솔로프리너)는 시간과 자원이 제한적이므로, 반복적인 업무를 자동화하여 효율성을 높이는 것이 중요합니다. Make.com을 활용하여 다양한 앱과 서비스를 연결하고 자동화 시나리오를 구축하여 업무 부담을 줄이고자 합니다.

# Task
다음 3가지 필수 시나리오를 Make.com으로 구축하기 위한 상세 설계를 제공하세요. 각 시나리오별로 다음 정보를 포함해야 합니다.

1.  **시나리오 개요**: 시나리오의 목적과 주요 기능에 대한 간략한 설명 (1~2 문장)
2.  **필수 모듈**: 시나리오 구축에 필요한 Make.com 모듈 목록 (각 모듈 이름과 역할 명시)
3.  **노드 연결 순서**: 각 모듈의 연결 순서를 번호로 명확하게 설명하고, 각 단계별 설정에 대한 핵심 가이드라인 제시 (예: 필터 설정 조건, 데이터 매핑 방법 등)
4.  **고려 사항**: 시나리오 운영 시 발생할 수 있는 문제점과 해결 방안 (예외 처리, 오류 알림 설정 등)

**필수 시나리오 3가지**:

*   **시나리오 1: 이메일 문의 관리 자동화**
    *   목적: 고객 문의 이메일을 자동으로 관리하고, 필요한 정보를 체계적으로 저장하여 응대 효율을 높임.
*   **시나리오 2: 소셜 미디어 콘텐츠 발행 자동화**
    *   목적: 미리 작성된 소셜 미디어 콘텐츠를 지정된 시간에 자동으로 발행하여 일관성 있는 브랜드 메시지를 전달하고 시간 절약.
*   **시나리오 3: 결제 정보 & 고객 관리 자동화**
    *   목적: Stripe 등의 결제 플랫폼에서 발생하는 결제 정보를 자동으로 Notion 데이터베이스에 저장하고 Slack 채널로 알림을 보내 빠른 고객 응대 가능하도록 함.

# Constraints
*   각 시나리오는 1인 사업자가 쉽게 이해하고 구축할 수 있도록 최대한 간결하고 명확하게 설명해야 합니다.
*   Make.com의 무료 플랜 또는 저렴한 플랜에서도 사용 가능한 모듈을 중심으로 설계해야 합니다. (비싼 유료 모듈 최소화)
*   각 모듈의 설정 가이드라인은 초보자도 쉽게 따라 할 수 있도록 구체적인 예시를 포함해야 합니다.
*   시나리오 운영 시 발생할 수 있는 오류를 최소화하고, 오류 발생 시 알림을 받을 수 있도록 설계해야 합니다.
*   각 시나리오 설명은 500자 내외로 제한합니다.

# Output Format

각 시나리오별로 다음 형식을 준수하여 상세 설계를 제공하세요.

```
## 시나리오 [번호]: [시나리오 이름]

### 1. 시나리오 개요:
[시나리오의 목적과 주요 기능을 간략하게 설명]

### 2. 필수 모듈:
*   [모듈 이름 1]: [모듈 역할 설명]
*   [모듈 이름 2]: [모듈 역할 설명]
*   ...

### 3. 노드 연결 순서:
1.  [모듈 이름 1]: [설정 가이드라인 (예: 필터 조건: 이메일 제목에 "문의" 포함)]
2.  [모듈 이름 2]: [설정 가이드라인 (예: 데이터 매핑: 이메일 주소 -> Notion 이메일 필드)]
3.  ...

### 4. 고려 사항:
*   [문제점 1]: [해결 방안]
*   [문제점 2]: [해결 방안]
*   ...
```
```

## 💡 사용 팁
1.  각 시나리오의 모듈 이름과 설정 가이드라인은 실제 사용하는 앱과 서비스에 맞게 변경해야 합니다. (예: Gmail -> Outlook, Notion -> Airtable)
2.  제공된 시나리오 외에도 반복적인 업무가 있다면, 이 프롬프트를 활용하여 자신만의 자동화 시나리오를 설계할 수 있습니다. `Task` 부분을 수정하여 원하는 자동화 목표를 명확하게 제시하세요.
3.  ChatGPT 또는 Gemini Pro 모델을 사용하여 이 프롬프트를 실행하는 것이 좋습니다. 필요에 따라 모델의 창의성 설정을 조절하여 더욱 상세하고 맞춤화된 결과를 얻을 수 있습니다.
